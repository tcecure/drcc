# Student Lifecycle Workflow

This document tracks the planned student lifecycle for DigitalRCC.

## Planned Flow

1. A prospective student requests access through the public portal.
2. Staff review the request and approve the appropriate role and program area.
3. The student signs in with Supabase Auth.
4. Approved training unlocks Moodle enrollment through a server-side integration.
5. Cyber range interest creates or updates a waitlist entry.
6. Lab provisioning requests flow through the internal bridge and are audited.
7. Student status, resources, and completion records remain visible in the dashboard.

## Sprint 0 Scope

Sprint 0 provides the routes, validation scaffolding, Supabase clients, middleware, and documentation needed for later workflow implementation.
